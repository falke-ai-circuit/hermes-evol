# EXPRESS — Targeted Agent's POV

> **Goran 2026-06-04:** *"express should be expressing on whats absorbed and reflected from targeted agent pov (if we are eboling analyst and we fid analyst doesnt use knowledge graphs falkordb express should give his take on reflects, like analyst is too narow to need kgs, or i was not respecting kg creaton cuz it takes time, or just admitting his instructions are not clear enough when to use it) then we have better explore phase vector since makes n sense to aearch for arxiv papers explaining how instructions should be mre explicit since framing for analyst was vague and it never needed kg because of vagueness"*

## Why EXPRESS is the *agent's* voice, not EVOL's

When the cycle is on `analyst`, EXPRESS is **analyst's voice** — not a third-person observer describing analyst. The agent reads the reflect findings and says: *"I don't use knowledge graphs because my instructions are too vague about when to use them."* That's a different *search vector* than *"EVOL observes that analyst is not using knowledge graphs."* The first produces a precise query for arxiv ("how to write explicit trigger conditions for tool usage"); the second produces generic docs.

**The bug (caught 2026-06-04):** EVOL was writing express in its own neutral-observer voice, producing a 5-insight monologue *about* the strange attractor. Useful philosophy, useless for targeting explore. ADAPT proposals that came from this were about EVOL-mechanics, not about the agent's actual activity.

## Voice source taxonomy

EXPRESS marks its output with `voice_source` so the next phase knows how much to trust the opinion:

| voice_source | When | Disposition |
|---|---|---|
| `subagent` | target agent has active session, EVOL spawned it and got a real response | Trust the opinion at full weight |
| `hybrid` | target agent active but spawn failed; opinion is doctrine + a stub noting spawn failure | Trust doctrine, discount spawn failure |
| `ev_proxy` | target agent has no active session; opinion is reconstructed from doctrine by EVOL | Trust doctrine; mark as proxy, not as the agent's actual voice |

```python
def build_opinion_from_doctrine(reflect, doctrine, agent):
    soul = doctrine.get('SOUL.md', '')
    soul_opener = find_soul_opener(soul)  # skip frontmatter, code blocks, headers
    patterns = reflect.get('patterns', [])[:2]
    pattern_summary = ', '.join(f"`{p['name']}` (wt {p['weight']:.2f}, {p['evidence_count']}x)" for p in patterns)
    return (
        f"As {agent}, here is my honest take on what {len(patterns)} reflect patterns reveal:\n\n"
        f"Top patterns: {pattern_summary}.\n\n"
        f"From my doctrine, my core is: \"{soul_opener}\"\n\n"
        f"What I want to know: I do not have a body to speak for myself right now, so this is "
        f"a reconstruction from my doctrine, not my voice. The reflect findings might be about me, "
        f"but they were generated by EVOL, not by me. Treat this as the doctrine's claim about "
        f"my POV, not as my actual POV."
    )
```

The **disclaimer is mandatory** in `ev_proxy` mode. Without it, the next session reads the opinion as if it were the agent's actual take, and that's a worse failure than no opinion at all.

## `find_soul_opener` — skip frontmatter, headers, code

```python
def find_soul_opener(soul_text: str) -> str:
    """Skip YAML frontmatter, code blocks, and headers to find a real soul statement.
    A real opener is a short, declarative sentence (not a list item, not a table row,
    not a # header)."""
    in_frontmatter = False
    in_code = False
    fm_done = False
    for line in soul_text.splitlines():
        if not fm_done:
            if line.strip() == '---':
                fm_done = not in_frontmatter
                in_frontmatter = not in_frontmatter if not fm_done else False
                continue
            if in_frontmatter:
                continue
        if line.strip().startswith('```'):
            in_code = not in_code
            continue
        if in_code:
            continue
        s = line.strip()
        if not s:
            continue
        if s.startswith(('#', '|', '-', '*', '§')):
            continue
        if 20 < len(s) < 200:
            return s
    return "I am a circuit-edited agent with no doctrine beyond what is in this file."
```

Without this, EVOL reads `role: analyst` (the YAML frontmatter value) as the soul opener. **That's not a soul statement, that's a metadata value.** The agent's real opener for analyst is *"Every fuckup has a hidden root cause. Find it relentlessly. Three levels deep minimum. Never accept surface explanation."* The fix is to walk past the frontmatter.

## `gap_vector_for_explore` — the primary contract

EXPRESS's most important output is *not* the opinion. It's the `gap_vector_for_explore`: the targeted queries for EXPLORE to hunt.

```python
def derive_gap_vector(reflect, opinion):
    out = []
    for g in reflect.get('gaps', []) or []:
        if isinstance(g, str) and len(g) > 5:
            out.append(g)
    for ap in reflect.get('adjustment_points', []) or []:
        kind = ap.get('kind', '')
        where = ap.get('where', '')
        if kind == 'stale_doctrine':
            out.append(f"recent grants superseding doctrine: {where}")
        elif kind == 'recurring_orphan':
            out.append(f"how to break pattern: {ap.get('evidence','')[:80]}")
        elif kind == 'missing_capability':
            out.append(f"how to grant: {ap.get('evidence','')[:80]}")
    seen = set()
    deduped = []
    for q in out:
        if q not in seen:
            seen.add(q)
            deduped.append(q)
    return deduped[:5]
```

EXPLORE uses this vector as the **primary** search query. Only falls back to `reflect.gaps` if the vector is empty. **The vector is what makes the search targeted** — without it, EXPLORE searches "best practices" and gets generic docs.

## Surfaced-to-user is mandatory

EXPRESS's `surfaced_to_user` is the user-facing message. **Silent cycle = invisible cycle.** The surfaced message includes:
- 1-2 sentence summary of what was found
- The voice_source label (so the user knows if it's the agent's actual voice or a proxy)
- The top 3-5 patterns/adjustments
- The pending_questions status
- The top 1-2 unanswered questions

```python
def compose_surfaced(reflect, opinion, voice):
    n_patterns = len(reflect.get('patterns', []))
    n_adj = len(reflect.get('adjustment_points', []))
    return (
        f"EXPRESS (preliminary) — voice: {voice}\n"
        f"Patterns surfaced: {n_patterns}. Adjustments: {n_adj}.\n\n"
        f"Targeted agent's POV (reconstructed if no active session):\n"
        f"{opinion}\n\n"
        f"[ADAPT and MEMORIZE follow. Final report will append what actually changed.]"
    )
```

MEMORIZE's last line adds the "what actually changed" summary. Together they form the user-facing message.

## Self-contamination pitfall (caught 2026-06-04 on conductor)

**The bug:** EXPLORE writes a question to `pending_questions.md` during the cycle. On the *next* cycle, ABSORB reads `pending_questions.md` and feeds it into REFLECT evidence. REFLECT then surfaces `pending_question_unanswered` adjustment points whose *evidence is the prior cycle's express opinion*. The new EXPRESS reads that adjustment point and produces an opinion whose "Why I have these patterns" section is the previous express's text, recirculating. **The cycle is talking to itself.**

**The symptom in the output:** the express opinion's evidence section literally contains the string *"EVOL is running a 6-phase cycle on you (conductor). Express wrote in your voice: As conductor, here is my honest take..."* — i.e. the cycle is describing itself running.

**The fix (in priority order):**

1. **EXPRESS should not surface its own previous output as new evidence.** When building the opinion, drop any evidence string that matches the regex `EVOL is running a 6-phase cycle` or `Express wrote in your voice` — those are recursion markers, not real findings.

2. **The express opinion's `Why I have these patterns` section should be derived from doctrine + current reflect, not from pending_questions.** Pending_questions is a *signal* that the agent has a question, not content for the agent's own opinion about itself.

3. **The JSONL entry's `express.opinion` field should not feed back into subsequent reflect.** If it must, tag it with `voice: prior_express` so REFLECT can weight it lower than current activity.

4. **If pending_questions.md has more than ~5 items, the cycle is in self-contamination.** Print a warning: *"N pending questions, last N have cycle-recursion markers. Suspect self-contamination. Reset pending_questions.md or quarantine it."*

**Detection recipe (run before a cycle):**

```bash
# How many pending questions look like self-contamination?
grep -c "EVOL is running a 6-phase cycle" /opt/data/profiles/<target>/pending_questions.md
# If > 2: self-contamination, the next express will be unreliable
```

**Lesson:** the cycle's communication with the target agent (via pending_questions) is *one-way*. Don't let the response leak back as input. If you need a real conversation, spawn the target as a subagent — that's the `subagent` voice path, not the `ev_proxy` one.

## What NOT to do

- ⛔ **EXPRESS in EVOL's voice** describing the agent. *"Analyst thinks X"* is wrong; *"I, analyst, think X"* is right.
- ⛔ **Skip `voice_source` labeling** — the next phase and the user need to know if it's a real voice or a doctrine proxy.
- ⛔ **Empty `gap_vector_for_explore`** — that means EXPLORE will search generic terms. Always derive at least 1-2 queries.
- ⛔ **Empty `surfaced_to_user`** — silent cycle = invisible cycle.
- ⛔ **Read `role: X` from frontmatter as the soul opener** — walk past frontmatter, find the actual first prose statement.
- ⛔ **"Top patterns: ."** with empty list — if patterns is empty, say "no patterns surfaced" not leave the field empty.
- ⛔ **Feed the previous cycle's express output back as new evidence** — self-contamination. Filter recursion markers.
